# Job Description Optimization Project Process Improvement Experience

**Date:** 2026-02-16
**Scenario:** User requests job description optimization

## Key User Feedback

User pointed out: Before providing a complete solution, should first ask for and provide existing materials:
1. Existing job description samples
2. List of positions needing optimization

## Problem Analysis

**Problems with Current Approach:**
- Directly providing generic solutions without targeting
- Not understanding first the specific format and problems of existing templates
- Not providing customized recommendations based on position types
- May lead to solutions disconnected from user's actual situation

**Improved Approach:**
- Add "material collection" step to information collection phase
- Review existing documents first, then provide targeted solutions
- If user doesn't have materials, clearly remind them of generic solution limitations

## Optimized Information Collection Process

### Step 1: Basic Information Understanding (Currently Done)
- Problem type (vague descriptions, outdated content, etc.)
- Number of positions (10)
- Position types (key positions)
- Whether templates exist (yes)

### Step 2: Material Collection (New Step) ⭐
**Should ask user:**
1. Can you provide existing job description samples (2-3 are enough)
2. Do you have a list of 10 key positions
3. If not, explain limitations and continue

**Standard Script:**
```
To provide a more precise solution, I'd like to understand some information first:

**Can you provide the following materials?**
1. Existing job description samples (2-3 are enough) - Help me understand current format and specific problems
2. List of 10 key positions - Help me understand position types and distribution

If you don't have these materials temporarily, I can first provide a generic methodology framework for reference.
Please note: Without specific materials, the solution will be quite generic and will need adjustment based on actual situation during implementation.
```

### Step 3: Targeted Solution Design
- Provide customized recommendations based on existing materials
- Provide solutions targeting specific problems
- Provide examples based on position characteristics

### Step 4: Complete Solution Output
- Provide highly actionable implementation plan
- Include optimization examples for specific positions
- Give actionable action items

## Experience Summary

**Core Principles:**
1. **Diagnose first, prescribe second** - Understand current situation first, then provide solution
2. **Material-driven precision** - Only with materials can you provide targeted recommendations
3. **Clear boundaries** - If no materials, clearly explain generic solution limitations

**Applicable Scenarios:**
- Job description optimization
- Organizational structure diagnosis
- Performance system design
- Compensation system optimization
- All HR-OD projects requiring "understand current situation first"

## Next Usage Reminder

When encountering similar requests, follow these steps:
1. Quickly understand basic information (3-5 questions)
2. Request existing materials (documents, data, org charts, etc.)
3. Explain: Have materials = precise solution, No materials = generic framework + risk warning
4. Provide customized solution based on materials
