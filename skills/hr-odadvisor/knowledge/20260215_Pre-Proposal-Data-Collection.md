# Pre-Proposal Data Collection Principle

**Topic:** Must ask if there are previous research records, org charts and other materials before outputting solution
**Record Date:** 2026-02-13
**Applicable Scenarios:** All solution design, consulting recommendations

---

## Core Principles

**Solutions must be grounded, first see what materials are available.**

**Previous research records, org charts, policy documents - these are more valuable than 10 interview questions.**

## Case Record: Weak Matrix Organization Collaboration Mechanism Optimization Solution

**Date:** 2026-02-13
**User Scenario:** 1000+ person traditional industry transformation enterprise, weak matrix structure, product-technology-operation departments have collaboration difficulties

---

## ❌ Serious Problem: Didn't Ask for Existing Materials After One-by-One Questions

### Problem Manifestation

**User Feedback:** "First, if you want to give solution after a few questions, then before giving solution, best to ask if there are previous research records or similar things, increase context, ensure solution is more grounded"

**Wrong Actions:**
1. **Collected information through one-by-one questions**: Asked size, industry, problem manifestations, matrix type, past attempts, decision layer determination, time rhythm and other 8 questions
2. **Last question was "whether there are existing materials"**: User answered "temporarily none"
3. **Immediately output complete solution**: Designed three-level collaboration mechanism solution based on interview information

**Error Reasons:**
- Although asked "whether there are existing materials", this was the **second to last question**
- Should ask in **early stage** of question flow, or as **final confirmation before transitioning to solution**
- Even without existing materials, should **clearly explain solution limitations**: "Designed based on interview information, can be further optimized if follow-up specific materials available"

**Why This Is Important:**
- **Previous research records**: May contain past failure experiences, key data, top management's real thoughts
- **Org charts**: Can see actual reporting relationships, department setup, position configuration
- **Policy process documents**: Can understand existing mechanism defects, avoid repeating recommendations
- **Data reports**: Can quantify problem severity, make solution more grounded

## ✅ Core Lesson: Must Ask "Whether There Are Previous Research Records" Before Outputting Solution

### Lesson 1: Ask Earlier in Timing

**Wrong Perception:** Treat "whether there are existing materials" as second to last question, ask then immediately give solution
**Correct Perception:** Should ask in **middle to late stage** of question flow, or as **special confirmation before outputting solution**

```
Suggested question sequence:
Questions 1-3: Background information (size, industry, development stage)
Questions 4-6: Problem specification (manifestations, frequency, impact scope)
Question 7: ★★★ Whether there are previous research records/org charts/policy documents? ★★★
         └─ If yes: Pause questioning, first read materials, then ask targeted questions
         └─ If no: Continue questioning, but clearly explain "solution based on interview information"
Questions 8-10: Past attempts, top management cognition, resource constraints
Step 11: Output solution (if no materials, explain limitations)
```

### Lesson 2: Even Without Materials, Explain "Can Optimize If Materials Available Later"

**Wrong Example:**
```
❌ "Are there existing materials?"
   User: "Temporarily none"
   Consultant: "OK, then I'll directly output solution..."
   → A month later user brings research report, find many parts of solution don't match
```

**Correct Example:**
```
✅ "Are there existing materials?"
   User: "Temporarily none"
   Consultant: "Understood. I'll design solution based on interview information. Also remind you:
       1. If you later have org chart, past research report, etc., can send to me anytime
       2. I'll further optimize solution based on these materials to make recommendations more practical
       3. Current solution is 'first draft based on interview information', not 'final version'"
   → User understands this is first draft, will be optimized based on materials later
```

### Lesson 3: Value of Previous Research Records Far Beyond Imagination

**Possible Materials:**
1. **Org Charts**: See actual reporting relationships, department setup, position configuration
2. **Past Research Reports**: Interview records, questionnaire data, problem analysis
3. **Policy Process Documents**: Performance assessment, recruitment process, promotion mechanism, etc.
4. **Data Reports**: Employee turnover rate, training coverage rate, cross-department collaboration satisfaction, etc.
5. **Past Reform Plans**: What was tried, why it failed, where key resistance lies
6. **Executive Meeting Minutes**: Decision layer's real thoughts, priorities, resource allocation

**Benefits of Having These Materials:**
- ✅ Avoid "reinventing the wheel": User may have already discussed similar solutions
- ✅ Avoid "hitting the wall": Know which methods were tried before but failed
- ✅ More precise solution: Design based on actual organizational structure and policies
- ✅ Save user time: Don't need to repeatedly explain background information

## ✅ Correct Approach: Standard Process for Material Collection

### Standard Question Script

**In middle to late stage of question flow (around 7-8th question), ask:**

```
Question X: Are there previous research records or existing materials?

**Please select the most applicable situation:**
A. Have complete past research/diagnosis reports
B. Have org charts, policy documents and other materials
C. Have some materials (please specify which)
D. Temporarily none

**Question Purpose:**
- If there are past research records, I can understand past failure experiences, key data, top management cognition
- If there are org charts, I can design more precise solutions
- If there are policy documents, I can avoid repeating recommendations for already existing content
- These materials are more valuable than 10 interview questions, can make solution more grounded
```

**If user has materials:**
```
Consultant: "Great! Please send materials to me, I'll read and analyze first.
       Based on these materials, I'll targetedly supplement a few questions, then design solution."
```

**If user doesn't have materials:**
```
Consultant: "Understood. No problem, I'll design solution based on interview information.
       Also remind you:
       1. If you later find org chart, past research report, etc., can send to me anytime
       2. I'll further optimize solution based on these materials
       3. Solution output now is 'first draft', not 'final version'"
```

## ⚠️ Prohibitions

**Prohibited before outputting solution:**
1. ❌ Don't ask "whether there are previous research records/existing materials"
2. ❌ Asked but didn't explain "can further optimize if materials available"
3. ❌ Even with materials, don't read, directly output solution based on interview information
4. ❌ After asking "whether there are materials", immediately output solution, don't give user time to prepare

**Prohibited in solution presentation:**
1. ❌ Pretend to have read materials user didn't provide (imagine content)
2. ❌ Clearly don't have materials, but write like "based on in-depth research" (dishonest)
