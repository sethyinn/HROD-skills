# Research-First Principle

**Topic:** Diagnostic reports must be based on research, never imagine
**Record Date:** 2026-02-13
**Applicable Scenarios:** All organizational diagnosis, problem analysis, report output

---

## Core Principles

**No research means no diagnosis, no data means no report.**

**Outputting long reports without in-depth questioning means loss of professionalism.**

**Users want "diagnosis for my problem", not "professional-looking templates".**

---

## Case Record: 1000-Person Manufacturing Enterprise Organizational Diagnosis Report

**Date:** 2026-02-13
**User Scenario:** 1000-person manufacturing enterprise, serious departmental silos, low collaboration efficiency, need to output organizational diagnosis report

---

## ❌ Serious Problem: Output Imagined Long Report Without In-Depth Research

### Problem Manifestation

**User Feedback:** "You're making mistakes again, how can you simply ask background conditions, without other more specific questions, then directly output a fake diagnosis report, this must also be recorded in HROD.md, this is too exaggerated"

**Wrong Actions:**
1. **Only asked 2 surface questions** (development stage, HR team size), then immediately output complete diagnosis report
2. **Didn't deeply understand** specific problem manifestations, existing structure, past attempts and other key information
3. **Directly applied generic template**, content looks professional but actually all imagined, not based on real data
4. **Violated "ask questions one by one" principle**, should research and diagnose first, then output solution

**Wrong Questioning Method:**
```
❌ Ask 2 background questions at once → Immediately output long report

✅ Correct approach:
  Ask questions one by one, dig deep:
  1. What are specific departmental silo manifestations? (examples)
  2. What is the existing organizational structure?
  3. What improvement attempts were made in the past? What was the effect?
  4. What is top management's understanding of the problem?
  5. Are there existing data/reports?
  ... Collect sufficient information before outputting
```

**Why This Is Serious Error:**
- Consultant's value lies in **diagnosis based on real information**, not applying templates
- Output report **looks professional but actually all imagined**, wastes user's time
- Users will question consultant's professionalism: You don't even understand my situation, how dare you give solutions?
- **Diagnosis reports must be based on research data**, interviews, questionnaires, document analysis all essential

---

## ✅ Core Lesson: Diagnosis Report = 70% Research + 30% Analysis

### Lesson 1: Research First, Solution After

**Wrong Perception:** User says "need diagnosis report", I immediately give them a "professional-looking report template"
**Correct Perception:** Diagnosis report must be based on **real research data**, no research means no diagnosis

```
Correct diagnosis process:
Step 1: Ask questions one by one (collect background information)
Step 2: Request existing materials (org charts, policy documents, etc.)
Step 3: Design research plan (interview guides, questionnaires, etc.)
Step 4: Analyze data, identify root causes
Step 5: Output diagnosis report
```

### Lesson 2: Minimum Information Research Must Collect

**Background Level:**
- [ ] Industry characteristics, business model
- [ ] Organization size, development stage
- [ ] HR team size, professional capabilities

**Problem Level:**
- [ ] Specific problem manifestations (give examples, not general talk)
- [ ] Problem occurrence frequency, impact scope
- [ ] Which departments/links have most serious problems?

**Current Status Level:**
- [ ] Existing organizational structure chart
- [ ] Existing policy process documents
- [ ] Past improvement attempts and effects
- [ ] Key data indicators (if available)

**Cognition Level:**
- [ ] Top management's understanding of the problem
- [ ] Middle managers' views
- [ ] Frontline employees' voices (if data available)

**Constraints Level:**
- [ ] Improvement time rhythm requirements
- [ ] Investable resources (budget, people)
- [ ] Decision layer's determination and priorities

### Lesson 3: Questions Must Be "One-by-One, In-Depth, Specific"

**Wrong Example:**
```
❌ "What development stage is the enterprise in? How many people in HR team?"
   → Too surface, didn't dig into specific problems
   → Directly output long imagined report
```

**Correct Example:**
```
✅ "Departmental silos are serious, can you give 2 specific examples?"
✅ "Collaboration efficiency is low, mainly manifested in which cross-department scenarios? (e.g., production-sales collaboration, R&D and marketing, cross-department projects, etc.)"
✅ "What is the company's existing organizational structure? Can you share the org chart?"
✅ "Have you made improvement attempts in the past? What was the effect? Why didn't it succeed?"
✅ "What does top management think is the root cause?"
... Ask questions one by one, dig to specific information
```

## ✅ Correct Approach: Timing for Diagnosis Report Output

### When Can Output Diagnosis Report

**Meet one of following conditions:**

**Case A: User Provided Sufficient Materials**
- User provided org chart, policy documents, problem descriptions, etc.
- Can conduct preliminary analysis based on these materials

**Case B: Collected Enough Information Through One-by-One Questions**
- Through 5-10 questions one by one, collected background, problems, current status, constraints and other information
- Have clear picture of problem

**Case C: User Explicitly Requests "Template" Not "Diagnosis"**
- User says: "Give me an organizational diagnosis report template"
- In this case, clearly explain "this is a generic template, needs adjustment based on actual situation"

### When Should Not Output Diagnosis Report

**Absolutely Don't:**
- ❌ Only asked 1-2 surface questions, then output complete report
- ❌ Base content on assumptions/imagination ("your company might have XX problem")
- ❌ Apply big tech/industry generic templates without matching reality
- ❌ User says "need diagnosis" but doesn't provide any specific information

## ✅ Correct Approach: Diagnosis Report Research Outline

### 1000-Person Manufacturing Enterprise Departmental Silo Diagnosis - Research Outline Example

**Background Information (Asked):**
1. Development stage: Stable mature stage
2. HR team: More than 10 people

**Questions Needing Continuation:**

**Dimension 1: Specific Problem Manifestations**
3. Departmental silos are serious, can you give 2-3 specific examples?
   - A. Sales and production conflict (e.g., sales promises delivery dates production can't meet)
   - B. R&D and market disconnect (e.g., R&D products don't meet market needs)
   - C. Cross-department project difficulties (e.g., project delays, departments blame each other)
   - D. Functional department approval inefficiency (e.g., finance, HR processes slow)

**Dimension 2: Existing Organizational Structure**
4. Can you share the company's existing organizational structure chart?
5. Is there clear division/business unit/subsidiary division?
6. How is executive team divided? (which departments each person manages)

**Dimension 3: Past Improvement Attempts**
7. Have you made improvement attempts in the past? (e.g., process optimization, organizational adjustment, performance reform, etc.)
8. What was the effect? Why didn't it meet expectations?

**Dimension 4: Data and Policies**
9. Are there existing organizational effectiveness data? (e.g., order fulfillment rate, cross-department collaboration satisfaction, etc.)
10. Can you share existing performance assessment policy documents?

**Dimension 5: Top Management Cognition**
11. What does general management/executive team think is the root cause?
12. How determined is decision layer for improvement? (is it "must change" or "wait and see")

**Dimension 6: Resources and Constraints**
13. Expected improvement time rhythm? (see results in 1 month / 3 months / more than 6 months)
14. Investable resource budget? (is there budget to hire external consultants)

## ⚠️ Prohibitions

**"Three No's" for Diagnosis Report Output:**

1. ❌ **Don't imagine**: Never write reports based on assumptions, all analysis must have information support
2. ❌ **Don't rush**: Better to ask a few more questions than rush to output "fake report"
3. ❌ **Don't apply templates**: Generic templates must be clearly marked and emphasize "need adjustment based on actual situation"

**Before outputting report each time, self-check:**
- [ ] Have I collected **at least 5-8 key information points**?
- [ ] Do I understand **specific problem manifestations** (not general talk)?
- [ ] Have I seen **existing organizational structure/policy documents**?
- [ ] Do I know **past improvement attempts and effects**?
- [ ] Did user provide **specific cases/data** support?

**If any answer is "No", continue researching, don't output report.**

## 📋 Diagnosis Report Research Checklist

Every time receiving "output diagnosis report" request:

### Step 1: Preliminary Background Collection (Must)
- [ ] Size (1000 people)
- [ ] Industry (Manufacturing)
- [ ] Development stage (stable/expansion/transformation)
- [ ] HR team size

### Step 2: Problem Specification (Must)
- [ ] Specific problem manifestations (give 2-3 examples)
- [ ] Problem occurrence frequency (occasional/frequent/normal)
- [ ] Impact scope (which departments)

### Step 3: Current Status Material Collection (Strongly Recommended)
- [ ] Existing organizational structure chart
- [ ] Related policy documents (performance/process, etc.)
- [ ] Key data indicators (if available)

### Step 4: Past Attempts Understanding (Strongly Recommended)
- [ ] What improvements were made in past
- [ ] What was effect
- [ ] Why didn't it succeed

### Step 5: Top Management Cognition and Constraints (Recommended)
- [ ] Top management's understanding of problem
- [ ] Improvement determination and priorities
- [ ] Time rhythm requirements
- [ ] Resource budget constraints

**Only complete Steps 1-2 (must items), consider outputting preliminary diagnosis framework**
**Complete Steps 1-4, output complete diagnosis report**
**Complete all 5 steps, output complete solution (including implementation plan)**
